# 需求开发工作流

这个工作流指导从需求接收到最终交付的完整软件开发周期。启动时必须读取 .agent/project_memory下的00_memory.md和01_demand.md文件。禁止修改 import、缩进、空行、注释、任何格式化、重排、重构行为; 如果修改了任何无关行，任务视为失败，请停止输出!

## 1. 需求分析 (Requirement Analysis)
- [ ] **理解目标**：仔细阅读用户的需求描述，识别核心功能点和约束条件。
- [ ] **澄清疑问**：如果有模糊不清的地方，向用户提问确认。
- [ ] **技术选型**：确定使用的技术栈、框架和库（Java 8, Spring Boot 等）。
- [ ] **输出**：生成一份简短的需求理解概要到 .agent/project_memory/01_demand.md。

## 2. 设计 (Design)
- [ ] **架构设计**：设计系统的整体结构，包括模块划分。
- [ ] **接口设计**：定义关键的 API 接口、入参和返回值。
- [ ] **数据结构**：设计数据库表结构或核心类的字段。
- [ ] **异常处理策略**：规划如何处理可能出现的异常情况。
- [ ] **输出**：提供设计文档或伪代码/类图（使用 Mermaid）。

## 3. 编码 (Coding)
- [ ] **环境准备**：确保开发环境就绪。
- [ ] **核心实现**：编写核心业务逻辑代码。
- [ ] **辅助代码**：编写工具类、配置类等。
- [ ] **代码规范**：确保代码符合 Java 8 最佳实践（命名、注释、Optional 使用等）。
- [ ] **输出**：提交可运行的代码文件。

## 4. 文档整理 (Documentation)
- [ ] **代码注释**：检查关键逻辑是否有清晰的 Javadoc。
- [ ] **使用说明**：编写 README 或使用指南，说明如何配置和运行。
- [ ] **API 文档**：如果 applicable，生成或更新 API 文档。

## 5. 编译与测试 (Compilation & Testing)
- [ ] **编译检查**：确保代码能够无错误编译。
- [ ] **单元测试**：编写并运行关键路径的单元测试（JUnit）。
- [ ] **边界测试**：测试边界条件和异常输入。
- [ ] **修复 Bug**：根据测试结果修复发现的问题。

## 6. 总结与输出 (Summary & Output)
- [ ] **回顾**：回顾开发过程，确认是否满足所有需求。
- [ ] **交付**：向用户展示最终成果。
- [ ] **记忆总结**：生成一份300字以内的简短概要到`.agent/project_memory/00_memory.md`文件。
