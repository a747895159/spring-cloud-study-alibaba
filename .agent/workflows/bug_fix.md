---
description: BUG 修复全流程：诊断 -> 修复 -> 记录日志
---

启动时读取.agent/project_memory/02_fix.md历史记录文件做参考。不涉及BUG的代码不要做任何改动，如果修改了任何无关行，任务视为失败，请停止输出;

1. **问题诊断**
   使用 `bug-diagnosis` 技能对当前问题进行深入分析。
   - 确认复现步骤
   - 定位代码位置
   - 分析根本原因

2. **方案制定与执行**
   - 制定修复方案（包含代码变更和测试计划）
   - 执行代码修复
   - 运行相关测试确保修复有效且无副作用

3. **记录修复日志**
   将本次修复的关键信息追加写入到 `.agent/project_memory/02_fix.md` 文件中。如果文件不存在则创建。
   
   **写入格式要求**：
   ```markdown
   ## [日期 时间] BUG修复记录
   
   ### 1. 问题现象
   [描述具体的错误表现]
   
   ### 2. 根本原因
   [解释导致 BUG 的核心原因]
   
   ### 3. 修复方案
   [简述修改了哪些逻辑或配置]
   ```
