# 文档编写器技能

## 用途
使用规定的模板（`docs/DOC_TEMPLATE.md`）和现有仓库内容来生成或刷新 `docs/planned/XX-*.md` 和 `docs/completed/XX-*.md` 文件。

## 使用说明
1. 检查相关源材料：
   - 阅读 `docs/DOC_TEMPLATE.md` 了解结构要求。
   - 从 `PLAN.md`、`AGENTS.md` 以及正在更新的任何现有任务文档中收集上下文。
   - 创建新任务时，识别现有编号并确保使用唯一的两位数前缀。
2. 按照精确的章节起草文档：元数据块、目标、前置条件/依赖项、实施步骤、验证、完成标准、备注/后续事项。
3. 通过链接相对路径来引用现有代码或文档，而非复制大段摘录。
4. 确认验收标准反映最新的仓库状态（测试、工具、环境要求）。
5. 使用命名约定 `NN-task-name.md` 将文件保存到正确的目录（`docs/planned/` 或 `docs/completed/`）。
6. 如果文件变更需要更新仓库，则运行必要的验证器（代码检查/测试）；记录执行的命令。
7. 在 PR/任务备注中汇总编辑内容。

## 验证
- 确保新建/更新的文档通过 Markdown 代码检查（如已配置）并符合 `docs/DOC_TEMPLATE.md` 结构。
- 确认编号序列保持唯一。
