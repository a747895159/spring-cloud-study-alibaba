# Skills 目录说明

## 📚 什么是 Skills？

Skills 是扩展 AI 助手能力的专业化指令集。每个 skill 定义了一个特定任务的执行流程、检查清单和最佳实践。

## 📁 目录结构

```
.agent/skills/
├── README.md
│
├── 后端开发 (4)
│   ├── java-code-review/          # Java 代码审查
│   ├── feature-implementation/    # 功能实现专家
│   ├── python-expert/             # Python 开发专家
│   └── java-diagram/              # Java 设计画图专家
│
├── 前端开发 (3)
│   ├── vue-frontend/              # Vue 前端开发
│   ├── ui-design/                 # UI 设计专家
│   └── frontend-design/           # 前端设计师
│
└── 通用技能 (3)
    ├── code-refactoring/          # 代码重构专家
    ├── bug-diagnosis/             # BUG 定位与修复
    └── design-document/           # 详细设计文档
```

## 🎯 可用技能

### 后端开发

#### 1. Java 代码审查 (`java-code-review`)
**用途**：对 Java 代码进行全面的质量审查

**检查维度**：
- ✅ 可维护性（命名、方法长度、职责划分）
- ✅ 异常处理（空 catch、资源释放、日志记录）
- ✅ 边界条件（空值检查、数组越界、数值溢出）
- ✅ 性能优化（循环优化、字符串拼接、集合选择）
- ✅ 并发安全（线程安全、锁使用、可见性）
- ✅ 安全性（SQL 注入、XSS、敏感信息）

**使用场景**：
- 代码提交前的自我审查
- Code Review 前的预检查
- 重构前的质量评估

#### 2. 功能实现专家 (`feature-implementation`)
**用途**：系统化地实现新功能，确保完整性和健壮性

**实施流程**：
- 📋 需求分析（功能边界、依赖识别）
- 🎨 设计方案（分层设计、接口设计）
- 💻 编码实现（参数校验、异常处理、日志记录）
- ✅ 完整性检查（边界条件、并发控制）

**使用场景**：
- 开发新功能
- 实现新接口
- 添加新业务逻辑

#### 3. Python 开发专家 (`python-expert`)
**用途**：Python 代码开发、审查和优化

**核心内容**：
- 🐍 代码规范（PEP 8、命名规范）
- 🐍 最佳实践（上下文管理器、列表推导式、装饰器）
- 🐍 常见模式（单例、装饰器、数据类）
- 🐍 性能优化（内置函数、集合查找、字符串拼接）
- 🐍 常见陷阱（可变默认参数、闭包变量）

**使用场景**：
- Python 代码开发
- 代码审查和优化
- 脚本编写

#### 4. Java 设计画图专家 (`java-diagram`)
**用途**：使用 Mermaid 可视化 Java 代码结构和交互

**核心能力**：
- 📊 类图（类结构、继承、依赖关系）
- 📊 时序图（方法调用流程、对象交互）
- 📊 流程图（业务逻辑、异常处理）
- 📊 状态图（状态流转）
- 📊 ER 图（实体关系）

**使用场景**：
- 分析现有代码结构
- 理解方法调用流程
- 梳理业务逻辑
- 设计数据模型
- 技术文档配图

### 前端开发

#### 5. Vue 前端开发 (`vue-frontend`)
**用途**：Vue.js 前端开发，包含组件设计和状态管理

**核心内容**：
- 📦 组件开发（Composition API、Props、Emit）
- 📦 状态管理（Pinia）
- 📦 路由配置（Vue Router）
- 📦 API 封装（Axios 拦截器）
- 📦 最佳实践（Composables、表单验证、性能优化）

**使用场景**：
- Vue 3 项目开发
- 组件设计和实现
- 状态管理和路由配置

#### 6. UI 设计专家 (`ui-design`)
**用途**：现代化 UI/UX 设计，创建美观易用的界面

**核心内容**：
- 🎨 设计原则（视觉层次、一致性、可访问性、响应式）
- 🎨 配色方案（主色调、中性色、深色模式）
- 🎨 字体系统（字体选择、大小层级、字重）
- 🎨 间距系统（4px/8px 基准网格）
- 🎨 组件设计（按钮、输入框、卡片、模态框）
- 🎨 动画效果（过渡、加载、微交互）

**使用场景**：
- UI 界面设计
- 组件样式设计
- 响应式布局

#### 7. 前端设计师 (`frontend-design`)
**用途**：创建具有卓越设计品质的独特、生产级前端界面

**核心内容**：
- 🎨 设计思维（目的分析、风格方向、差异化设计）
- 🎨 美学指南（排版、颜色、动效、空间构成、视觉细节）
- 🎨 风格系统（极简主义、复古未来、玻璃态、新拟态等）
- 🎨 技术实现（React/Vue/原生 HTML/CSS）
- 🎨 动画设计（CSS 动画、Framer Motion、滚动触发）
- 🎨 避免 AI 美学（独特字体、创意配色、个性化布局）

**设计原则**：
- ✅ 选择独特且有个性的字体（避免 Inter/Roboto/Arial）
- ✅ 明确的美学方向（极简、极繁、复古、奢华等）
- ✅ 精心设计的动画和微交互
- ✅ 不对称布局和创意空间构成
- ✅ 独特的视觉细节（渐变、噪点、玻璃态效果）
- ✅ 响应式设计和可访问性

**使用场景**：
- 构建着陆页、仪表盘、营销页面
- 创建 React/Vue 组件库
- 设计独特的 Web UI 界面
- 美化或重新设计现有界面
- 创建海报、素材等视觉元素

### 通用技能

#### 8. 代码重构专家 (`code-refactoring`)
**用途**：系统化地重构代码，提升可维护性和性能

**重构模式**：
- 🔧 提取方法（方法过长）
- 🔧 参数对象化（参数过多）
- 🔧 消除重复代码
- 🔧 简化条件表达式
- 🔧 引入策略模式
- 🔧 性能优化（N+1 查询）

**使用场景**：
- 代码重复过多
- 方法过长难以维护
- 性能需要优化
- 代码结构需要改进

#### 9. BUG 定位与修复 (`bug-diagnosis`)
**用途**：系统化地定位和修复 BUG

**诊断流程**：
- 🔍 问题理解（收集信息、分析错误）
- 🎯 问题定位（识别 BUG 类型、定位代码）
- 🧠 根因分析（5 Why 分析法）
- 🛠️ 修复方案（治本不治标）
- ✅ 验证与预防（测试验证、举一反三）

**使用场景**：
- 线上 BUG 修复
- 异常问题排查
- 性能问题定位
- 逻辑错误修复

#### 10. 详细设计文档 (`design-document`)
**用途**：编写清晰、完整的技术设计文档

**文档结构**：
- 📄 概述（背景、目标、范围）
- 📄 需求分析（功能需求、非功能需求）
- 📄 技术方案（架构、接口、数据模型、流程）
- 📄 关键设计（异常处理、性能、安全）
- 📄 测试与部署（测试方案、部署方案）
- 📄 风险评估（风险及应对措施）

**使用场景**：
- 新功能设计文档
- 系统重构设计
- 技术方案评审
- 团队技术沟通

## 🚀 如何使用

### 方式 1：直接请求
直接告诉 AI 使用某个技能：

```
请使用 Java 代码审查技能检查 IbdOrderManagerAppServiceImpl.java
```

### 方式 2：隐式触发
AI 会根据你的请求自动选择合适的技能：

```
帮我审查一下这个类的代码质量
```

```
这个 SQL 查询有性能问题吗？
```

## 📝 创建自定义技能

你可以创建自己的技能文件：

1. 在 `.agent/skills/` 下创建新文件夹
2. 创建 `SKILL.md` 文件
3. 按照以下格式编写：

```markdown
---
name: 技能名称
description: 技能描述
---

# 技能详细说明

## 目标
[说明这个技能要解决什么问题]

## 检查项/步骤
[列出具体的检查项或执行步骤]

## 输出格式
[定义输出报告的格式]

## 使用方法
[说明如何使用这个技能]
```

## 💡 最佳实践

1. **明确目标**：每个技能应该专注于一个特定领域
2. **结构化输出**：定义清晰的输出格式
3. **可执行性**：提供具体的、可操作的建议
4. **示例代码**：包含实际的代码示例
5. **持续更新**：根据项目需求不断完善技能

## 🔧 高级用法

### 组合使用多个技能
```
先用代码审查技能检查代码质量，然后用数据库优化技能检查 SQL 性能
```

### 自定义检查重点
```
使用 Java 代码审查技能，重点关注并发安全和异常处理
```
